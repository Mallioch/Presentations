Setting the Axis with flex-direction

Default display: block layout

Default behavior for these block elements is to take up 100% of their parent and stack on top of each other. Serious old-school behavior.

With a flex parent

Add a parent with display: flex and so many things change!

  • The parent element becomes the flex container.
  • Any direct child of the flex container becomes a flex item
  • Before they became flex items, their content size determined their height but not their width. When they become flex items and they are in a horizontal/row axis, content size determines both.
  • By default, flex containers have a horizontal, or row-based axis. In other words, it will line things up from beginning to end horizontally on a single row. In CSS property lingo, the container had a default of flex-direction: row applied.
